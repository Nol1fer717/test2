# Тест-кейсы для Task Tracker

## Описание
Данный документ содержит тест-кейсы для E2E тестирования таск-трекера Avito.

**URL тестируемого приложения:** https://avito-tech-internship-psi.vercel.app/issues

---

## 1. Создание задачи

### TC1: Успешное создание задачи с обязательными полями

**Предусловия:**
- Открыта страница списка задач
- Открыта форма "Создание задачи"

**Шаги:**
1. В поле "Название" ввести: `Оптимизация`
2. В выпадающем списке "Проект" выбрать `Оптимизация производительности`
3. В выпадающем списке "Приоритет" выбрать `High`
4. Убедиться, что поле "Статус" имеет значение `Backlog` и недоступно для редактирования
5. В выпадающем списке "Исполнитель" выбрать `Илья Романов`
6. Нажать кнопку "Создать"

**Ожидаемый результат:**
- Кнопка "Создать" становится активной после заполнения обязательных полей
- Задача успешно создаётся
- Пользователь видит подтверждение (редирект на список задач / карточку задачи / toast-уведомление)
- У созданной задачи: статус "Backlog", проект "Оптимизация производительности", приоритет "High", исполнитель "Илья Романов"

---

### TC2: Создание задачи со всеми заполненными полями

**Шаги:**
1. Ввести "Название": `Рефакторинг эндпоинтов`
2. Ввести "Описание": `Привести API к единому стандарту`
3. В поле "Проект" выбрать `Рефакторинг API`
4. В поле "Приоритет" выбрать `Medium`
5. Убедиться, что поле "Статус" `Backlog` и не кликабельно
6. В поле "Исполнитель" выбрать `Дмитрий Козлов`
7. Нажать "Создать"

**Ожидаемый результат:**
- Задача создается без ошибок
- Все введённые данные корректно сохраняются
- Статус задачи "Backlog"
- Данные корректно отображаются в системе после создания

---

### TC3: Проверка обязательности полей

**Шаги:**
1. Оставить поле "Название" пустым
2. Выбрать Проект: `Миграция на новую БД`
3. Выбрать Приоритет: `Low`
4. Выбрать Исполнитель: `Ольга Новикова`
5. Проверить состояние кнопки "Создать"

**Ожидаемый результат:**
- Кнопка "Создать" неактивна
- Задача не создаётся

---

### TC4: Проверка невозможности изменения статуса при создании задачи

**Шаги:**
1. Заполнить все обязательные поля валидными данными
2. Навести курсор на поле "Статус"
3. Попробовать открыть выпадающий список / кликнуть по полю

**Ожидаемый результат:**
- Поле "Статус" недоступно для взаимодействия
- Значение `Backlog` изменить невозможно
- Созданная задача имеет статус `Backlog`

---

### TC5: Создание задачи с любым доступным исполнителем

**Шаги:**
1. Ввести валидное Название
2. Выбрать Проект: `Переход на Kubernetes`
3. Выбрать Приоритет: `High`
4. В списке "Исполнитель" выбрать `Максим Орлов`
5. Нажать "Создать"

**Ожидаемый результат:**
- Выбранный исполнитель отображается в задаче
- Ошибок при сохранении нет
- Данные сохраняются корректно

---

### TC6: Проверка поведения кнопки "Создать"

**Шаги:**
1. Открыть форму создания задачи
2. Не заполняя поля, проверить кнопку "Создать"
3. Заполнить поля последовательно (по одному)
4. После заполнения последнего обязательного поля проверить кнопку

**Ожидаемый результат:**
- До заполнения всех обязательных полей кнопка неактивна
- После заполнения всех обязательных полей кнопка становится активной
- Клик по кнопке приводит к созданию задачи

## 2. Поиск задачи

### TC7: Поиск задачи по названию

**Предусловия:**
- Открыта страница со списком задач
- В списке есть задача с названием `Реализация темной темы`

**Шаги:**
1. В поле "Поиск" ввести `темной`
2. Дождаться обновления списка

**Ожидаемый результат:**
- В списке отображаются только задачи, в названии которых есть подстрока `темной`
- Отображаемые задачи соответствуют поисковому запросу
- Задачи без совпадений не отображаются

---

### TC8: Поиск задачи с применением фильтра по статусу

**Шаги:**
1. В поле "Поиск" ввести `Оптимизация`
2. В фильтре "Статус" выбрать `Done`
3. Дождаться обновления списка задач

**Ожидаемый результат:**
- Отображаются только задачи:
  - в названии которых есть `Оптимизация`
  - и статус которых `Done`
- Задачи с другим статусом и без совпадения по названию не отображаются

---

### TC9: Поиск задачи с фильтрацией по Доске

**Шаги:**
1. В поле "Поиск" ввести `загрузка`
2. В фильтре "Доска" выбрать `Оптимизация производительности`
3. Дождаться обновления списка

**Ожидаемый результат:**
- В списке отображаются только задачи проекта `Оптимизация производительности`
- Название каждой задачи содержит `загрузка`
- Задачи из других проектов не отображаются

---

### TC10: Поиск задачи с несколькими фильтрами одновременно

**Шаги:**
1. В поле "Поиск" ввести `Оптимизация`
2. В фильтре "Статус" выбрать `InProgress`
3. В фильтре "Доска" выбрать `Редизайн карточки товара`
4. Дождаться обновления списка задач

**Ожидаемый результат:**
- Отображаются только задачи, которые одновременно:
  - содержат `Оптимизация` в названии
  - имеют Статус `InProgress`
  - относятся к доске `Редизайн карточки товара`
- Если таких задач нет - отображается "Задачи не найдены"

---

### TC11: Поиск задачи без результатов

**Шаги:**
1. Ввести в поле "Поиск" строку `Несуществующая задача`
2. Оставить фильтры "Статус" и "Доска" в значении "Все"

**Ожидаемый результат:**
- Список задач пуст
- Пользователь видит сообщение вида "Задачи не найдены"

---

### TC12: Сброс фильтров и строки поиска

**Предусловия:**
- Отфильтровать список задач: ввести название существующего проекта в поле "Поиск"
- Выбрать Статус: `BackLog`
- Выбрать Доску: `Рефакторинг API`

**Шаги:**
1. Очистить поле "Поиск"
2. Установить фильтры Статус: "Все", Доска: "Все"

**Ожидаемый результат:**
- Отображается полный список задач без фильтрации

---

### TC13: Проверка чувствительности поиска к регистру

**Шаги:**
1. В поле "Поиск" ввести `ОПТИМИЗАЦИЯ`
2. Дождаться обновления списка

**Ожидаемый результат:**
- Найдены задачи, содержащие `Оптимизация` в любом регистре

---

## 3. Переход на доску проекта

### TC15: Переход на доску проекта со страницы "Проекты"

**Предусловия:**
- Открыта страница "Проекты"

**Шаги:**
1. Найти проект `Рефакторинг API`
2. Нажать кнопку "Перейти к доске" напротив проекта

**Ожидаемый результат:**
- Пользователь перенаправляется на доску проекта `Рефакторинг API`
- Отображается kanban-доска с колонками:
  - To Do
  - In Progress
  - Done
- В колонках отображаются задачи, относящиеся только к выбранному проекту
- Название проекта отображается в заголовке страницы

---

### TC16: Переход на доску проекта из карточки задачи

**Предусловия:**
- Открыта карточка задачи, относящейся к проекту `Редизайн карточки товара`

**Шаги:**
1. В карточке задачи нажать кнопку "Перейти на доску"

**Ожидаемый результат:**
- Пользователь перенаправляется на доску проекта `Редизайн карточки товара`
- На доске отображаются все задачи данного проекта
- Статус открытой ранее задачи соответствует колонке, в которой она отображается

---

### TC17: Проверка соответствия задач статусам на доске

**Шаги:**
1. Открыть доску любого проекта
2. Обратить внимание на задачи в колонках To Do, In Progress, Done

**Ожидаемый результат:**
- Каждая задача находится в колонке, соответствующей её статусу
- Отсутствуют задачи со статусом, не соответствующим колонке
- Нет задач из других проектов

---

## 4. Открытие карточки задачи

### TC18: Открытие карточки задачи из списка задач

**Предусловия:**
- Открыта страница "Список задач"

**Шаги:**
1. Найти задачу `Реализация новой галереи изображений`
2. Кликнуть по задаче (карточке / названию задачи)

**Ожидаемый результат:**
- Открывается модальное окно "Редактирование задачи"
- В карточке отображаются корректные данные:
  - Название
  - Описание
  - Проект
  - Приоритет
  - Статус
  - Исполнитель
- В карточке присутствует кнопка "Перейти на доску"

---

### TC19: Переход на доску проекта из карточки задачи, открытой из списка

**Шаги:**
1. Открыть карточку задачи из списка задач
2. Нажать кнопку "Перейти на доску"

**Ожидаемый результат:**
- Пользователь перенаправляется на доску проекта, к которому относится задача
- Открытая ранее задача присутствует на доске
- Задача находится в колонке, соответствующей её статусу

---

### TC20: Открытие карточки задачи с доски проекта

**Предусловия:**
- Открыта доска проекта

**Шаги:**
1. В любой колонке (To Do, In Progress, Done) выбрать задачу
2. Кликнуть по карточке задачи

**Ожидаемый результат:**
- Открывается окно "Редактирование задачи"
- Отображаются корректные данные задачи
- Кнопка "Перейти на доску" отсутствует, так как пользователь уже находится на доске проекта

---

### TC21: Редактирование задачи и проверка отображения изменений на доске

**Шаги:**
1. Открыть карточку задачи с доски проекта
2. Изменить статус задачи (например, из To Do в In Progress)
3. Нажать "Обновить"
4. Закрыть карточку

**Ожидаемый результат:**
- Карточка задачи перемещается в колонку In Progress
- Изменения сохраняются без ошибок
- Данные задачи обновлены корректно
